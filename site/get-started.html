Get started

Crucible is a Rust CLI (plus an MCP server and a phone-first dashboard) that reads and writes the same SQLite run ledger. This page is the exact command sequence Crucible's own cold-agent contract (SKILL.md) uses to onboard — nothing here is invented for the site.

Prerequisites

  • Rust 1.96 or newer (the workspace pins this as rust-version).
  • An OpenRouter API key if you want to run the live-model steps — everything through doctor works with zero spend and no key.

1. Clone the repo

git clone https://github.com/misty-step/crucible.git
cd crucible

2. Verify the install is actually live

Before running anything that spends real OpenRouter money, run the doctor. It proves the CLI runs, the MCP server initializes and lists its tools, crucible serve binds a port and answers /api/specs, and the SQLite run ledger can be created — all without a network call. 

cargo run -p crucible -- doctor --json

A broken check exits non-zero and names itself in the checks array. A missing OPENROUTER_API_KEY reports as warn, not fail — it only means the live-model steps below are skipped.

3. Validate a declared eval

The repo ships a small operator-facing benchmark you can validate and run immediately: 

cargo run -p crucible -- validate evals/operator-micro-benchmark-v0.json --json

4. Run it against two real models

cargo run -p crucible -- run evals/operator-micro-benchmark-v0.json \
  --models deepseek/deepseek-v4-flash,z-ai/glm-5.2 \
  --out runs/local/crucible-101/final \
  --db runs/local/crucible-101/final.sqlite \
  --json

5. Compare with a defensible verdict

runs compare reports a paired comparison with a Wilson interval and a noise-floor verdict — a rank gap this small comes back inside_noise_floor unless it is actually defensible. 

cargo run -p crucible -- runs compare \
  --benchmark operator-micro-benchmark-v0 \
  --left deepseek/deepseek-v4-flash \
  --right z-ai/glm-5.2 \
  --db runs/local/crucible-101/final.sqlite \
  --json

6. Open the dashboard

cargo run -p crucible -- serve \
  --db runs/local/crucible-101/final.sqlite \
  --specs evals \
  --port 4174

crucible serve reads the same run ledger the CLI just wrote to: the benchmark library, the run receipt, and the noise-floor comparison view.
